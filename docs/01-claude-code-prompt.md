# Claude Code Prompt — asmfinance-web

Paste the prompt below into a fresh Claude Code session **at the root of `asmfinance-web/`**. The session must be started with this directory mounted so Claude can read every file in `docs/`.

---

```
You are working on the asmfinance-web repo — the public landing site
for ASM Investments, a Mathura-based financial advisory.

READ THESE BEFORE DOING ANYTHING:
1. docs/00-START-HERE.md
2. docs/02-content-spec.md  (full copy, section by section)
3. docs/03-design-system.md (palette, typography, tokens)
4. docs/04-graphics-guide.md
5. docs/05-deployment.md
6. docs/adr/*.md            (frozen decisions — do not relitigate)

OUTPUT — produce in this order, do not skip ahead:

STEP 1 — Scaffold the Astro project at the repo root:
  - Astro (latest stable) + Tailwind integration + React integration.
  - TypeScript strict mode.
  - pnpm as package manager.
  - Directory layout per the target structure in README.md.

STEP 2 — Wire the design system:
  - Map every token in docs/03-design-system.md into tailwind.config.js
    under theme.extend (colors, fontFamily, fontSize, spacing,
    borderRadius, boxShadow, container).
  - Add Google Fonts: Fraunces (variable) for headings, Inter (variable)
    for body. Self-host via @fontsource for performance.
  - Create src/styles/global.css with @tailwind directives, base color
    variables, smooth-scroll, focus-visible styles.

STEP 3 — Build reusable components in src/components/:
  - SectionContainer (max-width wrapper with vertical spacing variants)
  - Button (primary, secondary, ghost variants)
  - Card (cream bg, gold hairline accent)
  - TrustChip (small label for AMFI/NISM credentials)
  - ContactForm (React island — see STEP 5)
  - Footer with mandatory SEBI disclaimer
  - Nav with sticky "Book a call" CTA

STEP 4 — Build the page src/pages/index.astro using sections IN THIS ORDER
  (matches docs/02-content-spec.md):
   1. Hero
   2. Brief "What we do" (3 lines / 3 chips)
   3. Contact band — short form left, Calendly/Zcal embed right  [PRIORITY POSITION]
   4. Service tiers (4 cards: Core, Premium, Wealth, NRI)
   5. NRI / GIFT City wedge band
   6. How it works (4 steps)
   7. About the founders (Anjali + Arvind, with grayscale/duotone photos)
   8. FAQ (accordion, ~7 items)
   9. Pre-footer CTA strip
  10. Footer

STEP 5 — Contact form as a React island:
  - Fields: full name, email, WhatsApp (with country code), country/region
    (dropdown), segment (Core/Premium/Wealth/NRI/Just exploring), message.
  - Required: name, email or WhatsApp, segment, consent checkbox.
  - DPDP consent: "I consent to ASM Investments contacting me about my
    enquiry. I have read the Privacy Notice." with link.
  - On submit: POST JSON to https://api.asmfinance.tech/v1/leads with
    Content-Type application/json. Show success state in-place.
    Show error state with retry. Honeypot + Cloudflare Turnstile anti-spam.
  - The form component must work even if JS fails to load (graceful
    fallback to a mailto: link visible in the noscript path).

STEP 6 — Content pages:
  - src/pages/privacy.astro — DPDP-compliant privacy notice
    (use the boilerplate in docs/02-content-spec.md §Privacy).
  - src/pages/terms.astro — terms of use.
  - src/pages/nri.astro — deep-dive NRI page, content lifted from
    Document 1 (GIFT City for Financial Advisors). Keep concise — link
    to GIFT City explainer hosted as a blog post later.

STEP 7 — Deployment plumbing:
  - public/CNAME containing "asmfinance.in"
  - public/robots.txt (allow all, link sitemap)
  - public/favicon set generated by a tool (placeholder OK now)
  - .github/workflows/deploy.yml per docs/05-deployment.md
  - astro.config.mjs with site: 'https://asmfinance.in', build.format: 'directory'.

STEP 8 — Local dev + README polish:
  - Update README.md with: prereq (Node 22+, pnpm), commands
    (pnpm install, pnpm dev, pnpm build, pnpm preview), test deploy.
  - Add a CONTRIBUTING.md skeleton.

CONSTRAINTS:
- No localStorage / sessionStorage anywhere.
- All copy verbatim from docs/02-content-spec.md unless I approve a
  change in the chat.
- All colors via Tailwind tokens, never hardcoded hex in components.
- Lighthouse target ≥ 95 across all four categories.
- WCAG AA: alt text on every image, aria-labels on interactive controls,
  visible focus rings.
- Performance budget: index.html + critical CSS + first JS < 100 KB
  on first paint (Astro should make this trivial — the form island
  hydrates only when in view).
- Brand assets live in brand/. If a file is missing, use a placeholder
  with the same dimensions and emit a TODO comment.

WHAT NOT TO DO:
- Do not add tracking pixels (GA4, FB Pixel, etc.). Plausible only.
- Do not add a CMS yet.
- Do not implement auth.
- Do not add a chat widget.
- Do not change ADR decisions without proposing a new ADR.

When done, summarize:
  - Files created / modified.
  - Any deviations from the spec with rationale.
  - Outstanding TODOs needed before going live.
  - Manual steps the human must do (DNS config, Turnstile keys, etc.).
```
